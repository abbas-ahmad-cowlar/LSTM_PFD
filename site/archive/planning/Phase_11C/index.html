    Decision 3: Statistical Analysis Framework¶

    Challenge: Users ask "Is Model A significantly better than Model B?"

    Solution: Statistical Testing Suite

    Tests Implemented:

    1. McNemar's Test (Paired, Binary)
    2. Use Case: Compare two models on same test set
    3. Null Hypothesis: Models have same error rate

    4. Output: p-value, conclusion ("Model A is significantly better, p=0.003")

    5. 5x2 Cross-Validation (More Robust)

    6. Use Case: Compare models with statistical rigor
    7. Method: 5 iterations of 2-fold CV

    8. Output: t-statistic, p-value, confidence interval

    9. Bootstrapping (Non-Parametric)

    10. Use Case: Estimate confidence interval for accuracy
    11. Method: Resample test set 1000 times

    12. Output: 95% CI (e.g., "Accuracy: 96.8% Â± 1.2%")

    13. Friedman Test (Multiple Models)

    14. Use Case: Compare 3+ models
    15. Output: Ranking, post-hoc pairwise comparisons

    Decision 5: Model Interpretation Dashboard¶

    Challenge: Transformer attention is complex, ResNet filters are numerous. Need systematic exploration tools.

    Solution: Model Introspection Suite

    Tools:

    1. Layer-by-Layer Activations
    2. Select: Signal + Model + Layer
    3. Display: Activation map for that layer

    4. Interaction: Scrub through layers like video timeline

    5. Filter Gallery (CNN/ResNet)

    6. Grid view: All filters in a layer
    7. Click filter: Show activations for that filter across dataset

    8. Purpose: Identify "what does filter #23 detect?"

    9. Attention Flow (Transformer)

    10. Animated visualization: How attention propagates through layers
    11. Slider: Scrub through time steps

    12. Heatmap: Which tokens attend to which

    13. Concept Activation Vectors (CAV)

    14. Define concept: "High-frequency bursts" (select 20 example signals)
    15. Train CAV: Linear classifier on activations
    16. Test CAV: Score any signal on "high-frequency-ness"

    17. Interpretation: "Model uses high-frequency bursts for Oil Whirl classification"

    18. Counterfactual Generator

    19. Input: Signal + Current prediction + Desired prediction
    20. Output: Minimal changes to flip prediction
    21. Example: "Change amplitude at 2.1-2.3s to flip from 'Normal' to 'Imbalance'"
