# Benchmark Protocol — Phase 4 (FROZEN upon ratification)

> **Status**: Draft for owner ratification (Convergence Plan P4.1).
> Once ratified, this protocol is **frozen**: any deviation is a dated
> amendment appended to §7 — never a silent edit. Runs that predate an
> amendment are kept and marked.

## 1. Task & data

- **Task**: 11-class fault classification on 1 s windows (20,480 samples).
- **Data**: `data/generated/dataset_v2.h5` (frozen, DVC-tracked; see
  `experiments/DATASET_V2.md`). Record-level splits as generated:
  12,320 / 2,640 / 2,640 windows (train/val/test) via `WindowedView`.
- **No peeking**: the clean `test` split and the `test_snr*` variants are
  evaluated exactly once per trained model, after training completes.
  Model selection uses `val` only.

## 2. Models & seeds

- **Tier 1 deep/physics nets (8)**: cnn1d, attention_cnn, cnn_lstm, resnet18,
  patchtst, hybrid_pinn, physics_constrained_cnn, multitask_pinn.
- **Classical (3)**: RandomForest, SVM, GradientBoosting on the 36-feature
  extraction (own script; same splits, features computed per window).
- **Ensemble (1)**: soft voting of the top-3 deep models by val accuracy.
- **Tier 2 (3)**: multi_scale_cnn, se_resnet18, signal_transformer — run
  **only** if the Tier-1 matrix completes with GPU time to spare.
- **Seeds**: 0, 1, 2 (extend to 5 only if all Tier-1 runs finish early; an
  extension covers ALL models or none — no cherry-picking).

## 3. Training budget (identical for every deep net)

| Setting | Value |
|---|---|
| Optimizer | Adam, lr = 1e-3 (the one documented exception: if a model's seed-0 run diverges — val acc < 20% at epoch 5 — retry that MODEL at lr = 3e-4 for ALL its seeds, recorded in §7) |
| Batch size | 64 |
| Max epochs | 60 |
| Early stopping | patience 10 on val accuracy |
| LR schedule | none (fixed lr) — schedulers are a per-model tuning axis we deliberately exclude for fairness |
| Loss | CrossEntropy (unweighted; classes are exactly balanced) |
| AMP | enabled on CUDA, disabled on CPU |
| Augmentation | none (DATASET_V2.md: clean provenance; revisit only as a documented amendment) |
| PINN metadata | operating-condition metadata (rpm/load/viscosity defaults) passed identically to all PINN variants |

## 4. Artifacts (per run — the runner enforces this)

`results/benchmark/deep/<model>/seed<k>/` containing:
- `metrics.json`: test accuracy, macro-F1, per-class accuracy, confusion
  matrix, full training history, and provenance (git SHA, data path, seed,
  host, device, wall time, config echo). The presence of this file marks the
  run COMPLETE — the queue skips it on resume.
- `best_model.pth`: best-val checkpoint (with optimizer state + history for
  resume). Checkpoints stay out of git (results JSONs/PNGs are committed).

## 5. Aggregation & statistics

- Table: mean ± std over seeds, per model: accuracy, macro-F1.
- Significance: Wilcoxon signed-rank on per-seed accuracies of best-physics
  vs best-vanilla; McNemar's test on pooled predictions for the headline pair.
- One significance-annotated bar figure + per-model confusion matrices.
- All generated by `scripts/compare_results.py` from `results/benchmark/`
  only — no hand-entered numbers anywhere.

## 6. Compute assignment

- **Office GPU PC**: the 8×3 deep matrix via `scripts/run_benchmark.py`
  (detached, resume-safe — see `experiments/OFFICE_PC_RUNBOOK.md`).
- **Laptop CPU**: classical baselines, ensemble assembly, aggregation.
- **Colab**: spillover lane (`scripts/colab/` batches) if the office GPU
  underdelivers; same artifact format, results merged by copy.
